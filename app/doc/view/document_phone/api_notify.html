异步通知

规则说明：

1、当用户完成支付，平台会通过商户下单时预留的回调连接notify_url，通知到下游商户，商户根椐订单完成后续的业务逻辑。
2、只有已完成的订单才会回调，其他状态不会回调。
3、如果回调成功，期望返回 “success” 字符串（订单处理过也返回“success”） 。返回其他则表示回调失败，平台会再次回调给商户（最多回调10次，每隔10秒回调一次）。
4、商户收到回调信息，需要进行验证签名，已保证信息安全。

请求方式：

提交方式：POST
Content-Type：application/json

回调参数：

参数名称 是否必填 字段类型 说明
mchid numeric 商户编号
order_no string 平台订单号
out_trade_no string 商户订单号
operator string 运营商
phone numeric 手机号
amount numeric 充值金额（元）
attach string 附加数据
success_time numeric 支付时间戳（年-月-日 时:分:秒）
status string NOTPAY：未充值；SUCCESS：已充值；
TIMEOUT：超时；ERROR：生成订单失败；
sign string 签名
